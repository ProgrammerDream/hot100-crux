# 临界点 · Hot100 刷题卡

把 **LeetCode Hot 100 题解里的易错临界点**（循环边界、指针初值、答案初值、二分区间…）做成**挖空卡**：给你一段正确的题解代码，只抠掉那个坑点，你把它填回去，翻开看「为什么对 / 为什么错」，然后按**间隔重复**把它刷成肌肉记忆。

**手机上像 App 一样用、且离线可用**——它是一个 PWA，不是原生 APK。

## 为什么是「挖空填回」而不是「二选一」

目标是「下笔不写错」（生成），不是「认出对的」（识别）。二选一会让你一半时间盯着错代码看、越练越混；挖空让你**全程读正确代码**，只在真正的临界点上做决策，训练任务和真实写代码对齐。

## 为什么是「封顶 Leitner」而不是 SM-2/FSRS

- 答对升一个盒子、答错打回盒 1，复习间隔**最长封顶 4 天**。
- 好处：卡片始终在手边循环，既不会像长间隔算法那样把卡推到 30 天后（那正是 iOS 清本地存储、又让你「今日 0 张」的元凶），也让工具保持每日习惯。
- 定位诚实：这是「把 Hot100 临界点刷到滚瓜烂熟」的**有限精通工具**，刷穿后定期回炉，不假装是无限内容的习惯 App。

## 核心设计

- **运行时不调大模型**：所有答案和解释在构建期写死进 `cards.js`，纯离线、零成本。
- **卡片锚定官方题解**：正确性来自 leetcode 官方题解，`cards.js` 是构建产物；卡 ≠ 题（有干净临界点才出卡，不凑满 100）。
- **进度只存本机**（localStorage）+ `navigator.storage.persist()` 抗清理；换机 / 清缓存前请在「☰ → 导出进度」做备份。
- **稳定卡片 ID**：进度以语义 ID 为主键，重建 / 新增卡不错行。
- **卡内报错**：翻开后点「卡有误」记录问题，可导出，喂给下一次构建修正。

## 本地运行

```bash
node server.js          # 零依赖，浏览器开 http://127.0.0.1:5510
```

手机同一 WiFi 下开 `http://<你电脑IP>:5510` 也能试；正式使用建议部署到 GitHub Pages（免费永久网址），手机打开后「添加到主屏幕」，此后彻底离线。

## 目录

```
index.html            单文件应用：挖空引擎 + Leitner 调度 + 持久化 + 导出导入
cards.js              卡库（构建产物，锚定 ../hot100project 官方题解）
sw.js                 Service Worker：外壳缓存，离线可用；改内容提升 CACHE 版本推更新
manifest.webmanifest  PWA 清单：添加到主屏、独立全屏
icon.svg              应用图标
server.js             本地静态服务器（零依赖）
scripts/build-cards.js  卡库构建脚本（从 problems.js 抽临界点，待接入）
```
